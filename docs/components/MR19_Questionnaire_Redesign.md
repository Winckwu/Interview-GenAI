# MR19: 元认知自我评估问卷重构方案

> **基于文献**: Schraw & Dennison (1994) MAI, Flavell (1979), 12-subprocess framework
> **问题识别**: 当前问卷设计不科学
> **重构目标**: 提供可靠、有效的元认知能力测量

---

## 📊 当前问卷的问题分析

### 1. **过于抽象和模糊**

#### 问题示例：
```
当前问题: "How well do you plan before starting a task?"
评分: 1-5 (Weak → Strong)

问题所在:
✗ "How well" 是主观感受,不是具体行为
✗ 缺少参照标准 - 什么叫"well"?
✗ 5分完全由个人理解决定
✗ 不同用户理解的"plan"含义不同
```

**为什么这有问题:**
- **Dunning-Kruger效应**: 能力弱的人往往高估自己(给自己打5分)
- **参照框架缺失**: 缺少具体行为锚点
- **文化差异**: 不同背景的人对"strong planning"理解不同

### 2. **缺少行为指标**

MAI量表的核心原则(Schraw & Dennison, 1994):
> **"测量可观察的行为习惯,而非主观自评能力"**

#### 对比分析:

| 类型 | 当前问卷 | MAI标准 |
|------|----------|---------|
| **描述方式** | "How well..." | "I do X when..." |
| **焦点** | 能力感知 | 具体行为 |
| **可验证性** | 无法验证 | 可通过行为验证 |
| **示例** | "How well do you plan?" | "I set specific goals before I begin a task." |

### 3. **维度覆盖不全**

当前仅4个问题(P/M/E/R各1题),而:
- **MAI**: 52题,8个子维度
- **项目12维度框架**: 需要P1-P4, M1-M3, E1-E3, R1-R2

**问题**:
- 单题无法可靠测量复杂维度
- 信度(reliability)低
- 无法捕捉细微差异

### 4. **李克特量表使用不当**

当前: 1-5 (Weak → Strong)

**文献标准**:
- **MAI**: True/False (是否有该行为习惯)
- **现代改进**: 1-5 (从不 → 总是)表示**频率**,而非能力

#### 为什么频率优于能力:
```
能力评估 (当前):
"我计划能力: 5分 (Strong)"
→ 无法验证,主观

频率评估 (标准):
"我在开始任务前总是设定具体目标"
→ 可通过行为日志验证
```

## 🔬 重构原则 (基于文献)

### 原则1: 行为化表述

❌ 错误: "How well do you monitor?"
✅ 正确: "I periodically review my comprehension as I work."

### 原则2: 具体情境化

❌ 错误: "How often do you check understanding?"
✅ 正确: "When working with AI, I stop every few minutes to ask myself if the output makes sense."

### 原则3: 可观察性

每个问题应该:
- 描述具体行为
- 可通过记录验证
- 避免抽象词汇

### 原则4: 充分覆盖

- 每个子维度 ≥ 2-3 题
- 总题数: 24-36题 (2-3题 × 12维度)
- 包含正向和反向题

## 📋 重构后的问卷设计

### **指导语**

```
以下陈述描述了人们与AI协作时的不同行为习惯。
请根据您的实际行为频率进行评分,而非您认为应该的频率。

评分标准:
1 = 从不 (Never)
2 = 很少 (Rarely) - 20%的时候
3 = 有时 (Sometimes) - 50%的时候
4 = 经常 (Often) - 80%的时候
5 = 总是 (Always) - 100%的时候

没有对错答案。请诚实作答。
```

### **Planning 规划 (P1-P4, 共12题)**

#### P1: 任务分解 (Task Decomposition) - 3题

**P1.1** (正向)
```
在让AI帮助我之前,我会先将任务分解为更小的步骤。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**P1.2** (正向, 具体化)
```
我会列出任务的子目标清单,然后逐个与AI协作完成。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**P1.3** (反向, 检验一致性)
```
我通常把整个任务直接交给AI,不做拆分。(反向计分)
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

#### P2: 目标设定 (Goal Setting) - 3题

**P2.1**
```
在开始与AI协作前,我会明确定义成功标准。
(例如: "输出应该包含X、Y、Z要素")
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**P2.2**
```
我会在提示词中明确说明我想要达到的具体目标。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**P2.3**
```
我对每个子任务都设定了清晰的质量基准。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

#### P3: 策略选择 (Strategy Selection) - 3题

**P3.1**
```
我会根据任务特性选择不同的提示策略。
(例如: 写代码用分步,写文章用创意)
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**P3.2**
```
我会考虑多种AI协作方式,选择最适合当前任务的方法。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**P3.3** (反向)
```
无论什么任务,我都用相同的方式与AI交互。(反向)
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

#### P4: 资源规划 (Resource Planning) - 3题

**P4.1**
```
我会明确划分哪些工作自己做,哪些让AI辅助。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**P4.2**
```
我坚持核心思考由自己完成,AI只负责辅助性工作。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**P4.3**
```
使用AI前,我会先尝试独立思考这个问题。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

### **Monitoring 监控 (M1-M3, 共9题)**

#### M1: 进度追踪 (Progress Tracking) - 3题

**M1.1**
```
在多步骤任务中,我会追踪"已完成到第几步"。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**M1.2**
```
我会定期检查当前进展是否符合预期时间线。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**M1.3**
```
当AI给出输出后,我会确认"这解决了我的子目标吗"。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

#### M2: 质量检查 (Quality Checking) - 3题

**M2.1**
```
我会逐行检查AI生成的代码/文本,而非直接使用。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**M2.2**
```
我会通过测试或交叉验证来确认AI输出的正确性。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**M2.3** (反向)
```
我通常假设AI的输出是正确的,不花时间验证。(反向)
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

#### M3: 上下文监控 (Context Monitoring) - 3题

**M3.1**
```
我会根据任务风险调整对AI的信任程度。
(高风险 → 低信任,低风险 → 高信任)
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**M3.2**
```
对于不同类型任务,我使用不同的验证严格程度。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

**M3.3**
```
我会意识到AI在某些任务上更可靠,在其他任务上需要更多监督。
1 (从不) - 2 - 3 - 4 - 5 (总是)
```

## 🎯 评分与解释系统

### 计算方法

```python
# 1. 反向题计分转换
reverse_items = ['P1.3', 'P3.3', 'M2.3']
for item in reverse_items:
    score[item] = 6 - score[item]  # 1→5, 2→4, 3→3, 4→2, 5→1

# 2. 计算12个子维度分数 (0-15分)
P1_score = (P1.1 + P1.2 + P1.3) / 3  # 平均分 1-5
P2_score = (P2.1 + P2.2 + P2.3) / 3
...
R2_score = (R2.1 + R2.2 + R2.3) / 3

# 3. 计算4个主维度分数 (0-20分)
Planning = (P1_score + P2_score + P3_score + P4_score) / 4  # 1-5
Monitoring = (M1_score + M2_score + M3_score) / 3
Evaluation = (E1_score + E2_score + E3_score) / 3
Regulation = (R1_score + R2_score) / 2

# 4. 映射到0-3分制(与行为分析对齐)
def map_to_0_3(score_1_5):
    """
    1.00-1.75 → 0分 (弱)
    1.76-2.75 → 1分 (中下)
    2.76-3.75 → 2分 (中上)
    3.76-5.00 → 3分 (强)
    """
    if score_1_5 < 1.76: return 0
    elif score_1_5 < 2.76: return 1
    elif score_1_5 < 3.76: return 2
    else: return 3

# 5. 总分 (0-36分)
total_score = sum([map_to_0_3(s) for s in all_subdimension_scores])
```

### 解释框架

#### 分数范围映射:

| 总分 | 等级 | 解释 |
|------|------|------|
| 0-9 | 低元认知 | Pattern F风险 - 需要全面干预 |
| 10-18 | 中低元认知 | Pattern B/C - 需要针对性支持 |
| 19-27 | 中高元认知 | Pattern A/D - 轻度优化 |
| 28-36 | 高元认知 | Pattern E - 保持现状 |

#### 单维度解释示例:

```
Planning (P) = 1.5/5 (低)

你的规划能力较弱。具体表现:
✗ P1 (任务分解): 1.8/5 - 很少将任务拆解为步骤
✗ P2 (目标设定): 1.2/5 - 缺少明确的成功标准
✓ P3 (策略选择): 3.5/5 - 能根据任务选择不同方法
✗ P4 (资源规划): 1.5/5 - 边界不清,过度依赖AI

🎯 建议:
- 使用MR1 (任务分解脚手架)
- 练习SMART目标设定
- 尝试"先独立思考5分钟"原则
```

## 🔍 效度验证计划

### 1. 构念效度 (Construct Validity)

**预期结构**: 12个子维度 → 4个主维度

**验证方法**:
- 验证性因子分析 (CFA)
- 预期拟合指标: CFI > 0.90, RMSEA < 0.08

### 2. 内容效度 (Content Validity)

**专家评审**:
- 5位元认知研究专家
- 评估题目是否准确测量目标维度

### 3. 效标效度 (Criterion Validity)

**并发效度**:
- 与行为数据相关性: r > 0.50
- 高分者应表现出更多元认知行为

**预测效度**:
- 高分者 → 更好的AI协作效果
- 低分者 → Pattern F风险

### 4. 信度 (Reliability)

**内部一致性**:
- Cronbach's α ≥ 0.70 (每个维度)
- Cronbach's α ≥ 0.85 (总量表)

**重测信度**:
- 2周后重测: r > 0.75
