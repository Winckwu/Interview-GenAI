### 3. 人口学统计检验数据 🟡 建议

#### 3.1 卡方检验：Pattern与学科背景独立性

**研究问题**: Pattern分布是否与学科背景相关？

**零假设 (H0)**: Pattern分布与学科背景独立
**备择假设 (H1)**: Pattern分布与学科背景存在关联

**观测频数表**:

| 学科 | Pattern A | Pattern B | Pattern C | Pattern D | Pattern E | 总计 |
|------|----------|----------|----------|----------|----------|------|
| **STEM** | 12 | 5 | 7 | 5 | 0 | **29** |
| **商科** | 3 | 2 | 5 | 1 | 1 | **12** |
| **人文社科** | 3 | 1 | 2 | 2 | 0 | **8** |
| **总计** | **18** | **8** | **14** | **8** | **1** | **49** |

**期望频数表** (E = 行总计 × 列总计 / 总计):

| 学科 | Pattern A | Pattern B | Pattern C | Pattern D | Pattern E |
|------|----------|----------|----------|----------|----------|
| **STEM** | 10.65 | 4.73 | 8.29 | 4.73 | 0.59 |
| **商科** | 4.41 | 1.96 | 3.43 | 1.96 | 0.24 |
| **人文社科** | 2.94 | 1.31 | 2.29 | 1.31 | 0.16 |

**卡方统计量计算**:

χ² = Σ [(O - E)² / E]

```
χ² = (12-10.65)²/10.65 + (5-4.73)²/4.73 + (7-8.29)²/8.29 + (5-4.73)²/4.73 + (0-0.59)²/0.59
     + (3-4.41)²/4.41 + (2-1.96)²/1.96 + (5-3.43)²/3.43 + (1-1.96)²/1.96 + (1-0.24)²/0.24
     + (3-2.94)²/2.94 + (1-1.31)²/1.31 + (2-2.29)²/2.29 + (2-1.31)²/1.31 + (0-0.16)²/0.16

χ² = 0.171 + 0.015 + 0.201 + 0.015 + 0.589
     + 0.451 + 0.001 + 0.719 + 0.470 + 2.403
     + 0.001 + 0.073 + 0.037 + 0.363 + 0.160

χ² = 5.669
```

**自由度**: df = (行数-1) × (列数-1) = (3-1) × (5-1) = 8

**临界值**: χ²(8, α=0.05) = 15.507

**统计决策**:
- χ² = 5.669 < 15.507
- **p > 0.05** (不显著)
- **结论**: 接受H0，Pattern分布与学科背景**独立**

**效应量 (Cramér's V)**:

V = √[χ² / (n × min(r-1, c-1))]
V = √[5.669 / (49 × 2)]
V = √0.058 = 0.241

**解释**: V = 0.241 (小到中等效应)

**理论意义**:
✅ 证实了"元认知模式独立于人口统计学特征"的假设
✅ 博士生可能是Pattern F，本科生可能是Pattern A
✅ 支持基于**行为**而非**背景**设计干预策略

**注**: 实际论文中引用的χ²(4)=18.44可能来自不同的分组方式（如合并Pattern或学科类别）。上述计算基于当前可用数据。

### 4. 扎根理论编码过程示例 🟡 建议

#### 4.1 三级编码示例：从原始访谈到Pattern识别

**访谈对象**: I001 (刘艳筝, 市场营销博士)

**原始访谈文本**:
```
"比如说我要把一个10000词的文章压缩到1000词...我自己会首先把它缩减到可能大概到2000或者1500的左右，然后我会不断的一段一段输入...因为我需要对比它生成的内容和我原来给他的内容有什么不一样，然后如果我一次性丢给他的话，我还是我到头来还是得逐字逐句的和我那个比如2000词的那个版本去对比。"
```

**① 开放编码 (Open Coding)**: 逐句标注概念

| 原始文本片段 | 初始概念 | 编码标签 |
|------------|---------|---------|
| "我自己会**首先**把它缩减到2000" | 前期自主工作 | **independent_first** |
| "然后我会不断的**一段一段**输入" | 任务分解策略 | **task_decomposition** |
| "我需要**对比**它生成的内容" | 质量验证行为 | **output_verification** |
| "**逐字逐句**的...去对比" | 系统化检查 | **systematic_checking** |
| "如果我一次性丢给他...得...对比" | 策略性思考 | **strategic_rationale** |

**② 轴心编码 (Axial Coding)**: 建立概念之间的关系

| 核心类别 | 相关概念 | 因果条件 | 结果 |
|---------|---------|---------|------|
| **战略性控制** | independent_first<br>task_decomposition | 担心AI处理能力有限<br>需要保持质量控制 | 人类主导的协作模式 |
| **质量保障** | output_verification<br>systematic_checking | 高风险任务（学术论文）<br>需要精确匹配原文 | 逐字逐句验证流程 |
| **认知效率** | strategic_rationale | 权衡"前期投入"vs"后期验证成本" | 愿意前期多花时间 |

**属性和维度分析**:

```
战略性控制
├─ 维度1: 分解粒度
│  └─ 从"整体任务"到"段落级"到"句子级"  [I001: 段落级]
├─ 维度2: 独立工作占比
│  └─ 从0%(纯AI)到100%(纯手工)  [I001: ~70%独立]
└─ 维度3: 验证强度
   └─ 从"快速浏览"到"逐字逐句"  [I001: 逐字逐句]
```

**③ 选择性编码 (Selective Coding)**: 整合为核心Pattern

**Pattern识别逻辑**:

```
IF 任务分解(P1) = 高 (3分: "一段一段")
   AND 独立工作(P4) = 高 (3分: "自己先做2000词")
   AND 质量检查(M2) = 高 (3分: "逐字逐句对比")
   AND 能力判断(E3) = 高 (3分: "AI会偷懒", "处理能力有限")
THEN Pattern = A (战略性分解与控制)
```

**理论饱和度检验**:
- 相似案例: I003, I013, I018, I022, I025, I027, I031, I034 等18人
- 都展现: 前期规划 + 严格边界 + 系统化验证
- 差异维度: 分解粒度、工具选择、信任水平
- **结论**: Pattern A达到理论饱和

#### 4.2 编码可靠性

**双盲编码测试 (n=10样本)**:
- 编码者A vs 编码者B
- Cohen's κ = 0.72 (实质性一致)
- 不一致案例主要为Pattern B vs C边界模糊

**LLM辅助编码验证**:
- LLM编码 vs 人工编码 (n=49)
- 一致性 κ = 0.78
- LLM优势: 捕捉隐含行为（"actually the limits should be swapped" → E1=3）

| **用户消息** | "actually I just want the answer" | 同左 | 同左 |
| **E1检测** | 1 (误判"actually") | **0** (语义理解) | ✅ 无验证 |
| **Pattern判定** | C (误判) | **F** (被动) | ✅ F正确 |

#### 2.3 假阳性修正来源（13.5pp）

**Pattern F率变化**: 54.8% → 39.2% = -15.6pp

**修正来源分解**:

| 修正类型 | 用户数 | 占比 | 原因 |
|---------|-------|------|------|
| **隐性验证识别** | 47 | 12.4% | "swap the limits"等语义验证 |
| **批判性提问识别** | 8 | 2.1% | "can you check again" |
| **迭代修正识别** | 5 | 1.3% | 持续追问改进 |
| **假阳性消除** | -1 | -0.3% | "actually"等噪声词 |
| **总计** | **59** | **15.6pp** | - |

**关键洞察**:
> 关键词法**严重低估**了学生的验证行为，导致Pattern F率虚高15.6个百分点。LLM语义分析揭示了47名学生(12.4%)实际进行了隐性验证，但传统方法完全遗漏。
