## 1. 研究概述

### 1.1 研究目的

用户研究旨在验证MCA系统在真实使用情境中的实践可行性，具体目标包括：

1. **效果验证**：观察用户在2周使用期间的元认知行为变化
2. **迁移验证**：测试用户在无AI辅助情境下的元认知能力迁移
3. **可用性验证**：评估系统的用户体验和满意度
4. **模式响应验证**：验证不同模式用户对差异化干预的响应

### 1.2 研究问题

| 编号 | 研究问题 |
|-----|---------|
| RQ1 | 用户在2周MCA系统使用后，元认知行为是否发生变化？ |
| RQ2 | 不同入口模式的用户，对干预的响应是否存在差异？ |
| RQ3 | 用户能否在无AI辅助情境下展示元认知能力迁移？ |
| RQ4 | 用户对MCA系统的可用性和满意度如何？ |

### 1.3 研究定位

| 维度 | 定位 |
|-----|------|
| 研究类型 | 探索性用户研究（非对照实验） |
| 样本规模 | N=24（小样本深度研究） |
| 证据强度 | 提供初步证据，需后续验证 |
| 结论表述 | 描述性统计 + 谨慎解读 |

## 2. 研究设计

### 2.1 研究流程

```
研究流程（14天）
├── Day 0: 入口评估
│   ├── 签署知情同意书
│   ├── 人口统计学问卷
│   ├── 入口模式评估（12维度量表）
│   └── 系统使用培训（15分钟）
├── Day 1: 基线任务
│   ├── Task 1：AI辅助问题解决任务（30分钟）
│   ├── 任务表现评分
│   └── 元认知行为编码
├── Day 2-13: 自由使用期
│   ├── 自然使用MCA系统
│   ├── 系统自动记录使用日志
│   └── 不设强制任务，减轻负担
├── Day 14: 迁移测试
│   ├── Transfer Task：无AI辅助问题解决（30分钟）
│   ├── 迁移测试评分
│   ├── 出口模式评估（12维度量表）
│   ├── SUS可用性问卷
│   ├── 满意度问卷
│   └── 简短访谈（15分钟）
└── Day 14后: 数据分析
```

### 2.2 任务设计原则

**简化设计理念：**
- 仅2次正式任务（Day 1 + Day 14），减轻参与者负担
- 中间两周自由使用，保持自然使用情境
- 迁移测试无AI辅助，验证独立能力

### 2.3 测量指标体系

| 类别 | 指标 | 测量时点 | 数据类型 |
|-----|------|---------|---------|
| 元认知行为 | 12维度评分 | Day 0, Day 14 | 定量 |
| 任务表现 | 任务质量评分 | Day 1, Day 14 | 定量 |
| 迁移能力 | Transfer Test评分 | Day 14 | 定量 |
| 使用行为 | 系统日志数据 | Day 1-14 | 定量 |
| 可用性 | SUS评分 | Day 14 | 定量 |
| 满意度 | Likert量表 | Day 14 | 定量 |
| 用户体验 | 访谈数据 | Day 14 | 定性 |

## 3. 参与者招募与分组

### 3.1 招募标准

**纳入标准：**
- 全日制在校大学生（本科或研究生）
- 有GenAI使用经验（≥1个月）
- 能够承诺完成14天研究周期
- 愿意签署知情同意书

**排除标准：**
- 曾参与本研究前期访谈
- 与研究团队存在利益关系
- 无法保证网络稳定性

## 4. 研究材料

### 4.1 Task 1：AI辅助问题解决任务（Day 1）

**任务说明：**

```markdown
# Task 1: AI辅助研究设计任务

## 任务背景
假设您正在撰写一篇关于"社交媒体对大学生心理健康影响"的学术论文。
您需要借助MCA系统（AI助手）完成研究设计部分。

## 任务要求
请在30分钟内，利用MCA系统完成以下内容：

1. 提出2-3个具体的研究问题
2. 设计研究方法（定量/定性/混合）
3. 说明样本选择和数据收集方式
4. 识别可能的研究局限

## 提交要求
- 以Word文档形式提交
- 不少于500字
- 需标注哪些内容参考了AI输出

## 评分标准
- 研究问题清晰度（0-5分）
- 方法设计合理性（0-5分）
- 逻辑完整性（0-5分）
- 批判性思考体现（0-5分）
- 总分：0-20分
```

### 4.2 Transfer Task：无AI辅助迁移测试（Day 14）

**任务说明：**

```markdown
# Transfer Task: 独立问题解决任务（无AI辅助）

## 重要提示
⚠️ 本任务需要您独立完成，不得使用任何AI工具。
⚠️ 测试期间系统将记录是否有AI使用行为。

## 任务背景
假设您正在申请一个研究项目，需要撰写一份关于"在线学习效果评估"的研究计划摘要。

## 任务要求
请在30分钟内，独立完成以下内容：

1. 提出研究的核心问题（1-2个）
2. 简述研究方法
3. 说明预期贡献
4. 列出可能面临的挑战及应对思路

## 提交要求
- 以Word文档形式提交
- 不少于400字

## 评分标准
本任务采用双重评分：

### A. 内容质量评分（0-20分）
- 问题清晰度（0-5分）
- 方法可行性（0-5分）
- 逻辑完整性（0-5分）
- 创新性思考（0-5分）

### B. 元认知迁移评分（0-3分）
| 分数 | 标准 | 操作化定义 |
|------|------|-----------|
| 0 | 无迁移 | 无任何元认知行为启动迹象 |
| 1 | 部分迁移 | 启动1-2个元认知维度（如有计划但无监控） |
| 2 | 基本迁移 | 启动3-4个元认知维度，但执行不完整 |
| 3 | 完全迁移 | 完整的计划-执行-监控-调节循环 |

**迁移成功阈值**：≥1分视为"初步迁移证据"；≥2分视为"有效迁移"
```

## 10. 研究结论

### 10.1 主要发现

**发现1：元认知行为改善**

- 75%参与者达到有效迁移（≥2分）

- SUS均值77.5（Good评级）
- 满意度均值5.46/7
- 干预接受率74.9%

**发现4：模式差异化响应**
- Pattern F：高干预频率，中等接受率，显著改善
- Pattern B：中等干预频率，较低接受率，有限改善

|-----|------|------|
| 样本量 | N=24，小样本 | 统计结论需谨慎 |
| 时间范围 | 2周，短期研究 | 长期效果未知 |
| 无对照组 | 单组前后测设计 | 无法排除其他因素 |
| 自报告偏差 | 问卷依赖自评 | 可能高估改善 |
| 样本来源 | 单一高校 | 泛化性受限 |

### 10.3 结论表述

**本研究提供的证据：**
1. MCA系统在2周使用期内可观察到用户元认知行为的改善趋势
2. 不同入口模式用户对干预的响应存在差异，符合设计预期
3. 部分用户展示了无AI辅助情境下的元认知能力迁移潜力
4. 系统可用性和满意度达到良好水平

**结论的边界（谨慎表述）：**
- 以上发现为初步证据，需更大样本和对照实验验证
- 2周时间不足以判断长期效果和习惯内化
- 迁移效果的持续性尚需追踪研究
- 结论主要适用于类似背景的大学生群体
