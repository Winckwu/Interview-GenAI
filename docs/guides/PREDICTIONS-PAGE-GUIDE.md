# Predictions 页面完整指南

## 🎯 什么是 Predictions 页面？

**Predictions 页面**是一个帮助你**预测和验证自己AI使用方式**的工具。

就像天气预报一样：
- 🌤️ **预报**：你预测"今天会下雨"
- 🌧️ **验证**：今天实际下雨了→预报正确 ✅
- 🌞 **记录**：追踪你的预报准确率

---

## 📚 核心概念

### 1. 你在预测什么？

你在预测：**在执行某个任务时，你会使用哪种 AI 使用模式**

**6 种可能的模式：**
| 模式 | 说明 | 特点 |
|------|------|------|
| **A** | 战略分解与控制 | 最优秀：任务分解 + 高验证 + 保持技能 |
| **B** | 迭代精化与校准 | 很好：通过快速迭代学习 |
| **C** | 上下文敏感适应 | 很好：根据任务动态调整 |
| **D** | 深度验证与批判 | 很好：系统性怀疑 + 高验证 |
| **E** | 教育学反思与自监控 | 很好：AI当教师，定期不用AI |
| **F** | ⚠️ 低效被动使用 | 危险：低验证 + 无脑接受 |

### 2. 任务相关信息

在创建预测时，你需要提供：

**任务类型** (Task Type)：
- **Coding**：编程任务
- **Writing**：写作任务
- **Analysis**：分析任务
- **Design**：设计任务

**复杂度** (Complexity)：
- **Easy**：简单任务（30分钟以内）
- **Medium**：中等任务（30分钟到2小时）
- **Hard**：困难任务（2小时以上）

**时间约束** (Time Constraint)：
- **Limited**：时间紧张（赶截止日期）
- **Adequate**：时间充足（有足够思考空间）
- **Abundant**：时间充裕（可以反复验证）

**是否有AI可用** (AI Available)：
- **是 (True)**：可以使用AI
- **否 (False)**：不能使用AI（离线环境）

## 🚀 如何使用？

### 步骤 1️⃣: 打开 Predictions 页面

在导航菜单中找到 **"Predictions"** 链接
- 通常在左侧导航栏或顶部菜单

### 步骤 2️⃣: 查看你的统计数据

页面顶部会显示：

```
┌─────────────────────────────────┐
│ 📊 你的预测统计                  │
├─────────────────────────────────┤
│ Total Predictions  : 5 个        │
│ Accuracy           : 80%         │
│ Correct            : 4 个        │
│ Pending Feedback   : 1 个        │
└─────────────────────────────────┘
```

**说明：**
- **Total Predictions**: 你创建了多少个预测
- **Accuracy**: 准确率（正确预测 ÷ 总预测 × 100%）
- **Correct**: 已验证正确的预测数
- **Pending Feedback**: 还没验证的预测数

### 步骤 3️⃣: 创建新预测

点击 **"+ New Prediction"** 按钮

会弹出一个表单：

```
┌─────────────────────────────────┐
│ Create New Prediction           │
├─────────────────────────────────┤
│ Task ID        : [输入框]        │
│ Task Type      : [Coding ▼]     │
│ Complexity     : [Medium ▼]     │
│ Time Constraint: [Adequate ▼]   │
│ AI Available   : [✓ 是]         │
│ [Submit Prediction]             │
└─────────────────────────────────┘
```

**填表指导：**

#### Task ID
- 输入任务的唯一ID
- 例如：`task-001`, `project-v2-auth`, `writing-essay-01`
- 这样你以后能找到对应的任务

#### Task Type
- 选择这个任务的类型
- Coding：写代码
- Writing：写文章、报告、邮件
- Analysis：分析数据、文档
- Design：设计UI、架构

#### Complexity
- 评估任务的难度
- Easy：简单，你很熟悉
- Medium：中等，需要思考
- Hard：困难，需要深入思考

#### Time Constraint
- 评估你有多少时间
- Limited：赶时间（1小时以内）
- Adequate：时间充足（几小时）
- Abundant：时间很多（可以反复思考）

#### AI Available
- 勾选表示可以用AI
- 不勾表示离线或不用AI

### 步骤 4️⃣: 做出预测

**现在你要预测**：在这个任务中，**你会表现出哪种模式 (A-F)?**

根据你对自己的了解：
```
📝 在做一个 Medium 复杂度的编程任务，时间充足：

"我会表现出 Pattern B
因为：
- 我会尝试自己先解决
- 如果卡住，用AI帮助
- 解决后再验证逻辑
- 这样边做边学，效率高"
```

**提示**：
- 如果你平时很谨慎、都验证 → 可能是 A/D/E
- 如果你快速迭代、失败了再试 → 可能是 B/C
- 如果你时间紧、直接用AI答案 → 可能是 F ⚠️

### 步骤 5️⃣: 提交预测

点击 **"Submit Prediction"** 按钮
- ✅ 预测被保存
- ✅ 会出现在历史列表中，状态是 "Pending"

### 步骤 6️⃣: 完成任务 → 验证预测

**当你完成这个任务后：**

1. **自己记录**：你在这个任务中实际表现出的模式是什么？

2. **在系统中更新**（当功能完成时）：
   - 在 Data Browser 或相关页面提交反馈
   - 标记：这个预测是否准确

3. **系统会自动计算**：
   - 这个预测是否正确（✅ 或 ❌）
   - 你的整体准确率

## 📊 预测历史表格

表格显示你所有的预测：

```
┌────────┬──────────────┬───────────┬────────┬────────┬──────────┐
│Task ID │Predicted Pat │Actual Pat │Confid │Status  │Date      │
├────────┼──────────────┼───────────┼────────┼────────┼──────────┤
│task-1  │A             │A          │95%    │✅Correct│2025-11-18│
│task-2  │B             │B          │87%    │✅Correct│2025-11-17│
│task-3  │C             │A          │72%    │❌Wrong  │2025-11-16│
│task-4  │B             │-          │81%    │⏳Pending│2025-11-15│
└────────┴──────────────┴───────────┴────────┴────────┴──────────┘
```

**列说明：**

| 列名 | 含义 |
|------|------|
| **Task ID** | 任务编号 |
| **Predicted Pattern** | 你预测的模式 (A-F) |
| **Actual Pattern** | 实际观察到的模式（还没验证时是 `-`） |
| **Confidence** | 系统的置信度（你的预测有多确定） |
| **Status** | 验证状态 ⏳ Pending / ✅ Correct / ❌ Incorrect |
| **Date** | 创建日期 |

## 💡 使用场景

```

2. 完成任务：
   - 实际表现：Pattern D ✅
```

```
   - 准确率：你对自己了解有多深？
```

## 🎯 为什么要做预测？

### 好处 1️⃣: 了解自己
- 你真的了解自己如何使用AI吗？
- 预测 vs 实际 = 你的自我认知准确度

### 好处 2️⃣: 建立意识
- 做预测前你会**思考**："我在这个任务中会怎么做？"
- 这种**元认知**（thinking about thinking）很重要

### 好处 3️⃣: 改进行为
- "我预测会过度依赖AI"
- → 这让你在任务中**更警觉**
- → 你实际表现会**更好**

### 好处 4️⃣: 数据驱动
- 准确率：我对自己了解有多深？
- 趋势：我在改进吗？
- 模式：不同任务中我表现不同吗？

## 📈 解读统计数据

### 准确率的含义

```
准确率 = 正确预测 ÷ 总预测 × 100%

95% → 你对自己了解很深（几乎总能预测对）
70% → 你对自己有基本了解
50% → 你对自己理解一般（有时出其不意）
```

### 什么是好的准确率？

| 准确率 | 说明 | 行动 |
|--------|------|------|
| **90%+** | 🌟 非常高 | 保持这样，你很了解自己 |
| **70-90%** | ✅ 很好 | 继续做预测，逐步提升 |
| **50-70%** | 🤔 一般 | 做更多预测，了解自己 |
| **<50%** | ⚠️ 需要思考 | 你可能误解了模式定义 |

## ❓ 常见问题

### Q1: 预测如何提交反馈验证？
**A**:
- 当前版本：在 Data Browser 页面提交
- 未来版本：预计在 Predictions 页面直接点击预测来更新

### Q2: 为什么我的预测总是错？
**A**:
- 可能你对 6 种模式理解还不够深入
- 建议：读一遍 Patterns Page 的说明
- 或者：实际完成任务时，更仔细地观察自己的行为

### Q3: 预测有具体的"答案"吗？
**A**:
- 没有绝对答案
- 根据**你的实际表现**判断
- 自我观察很重要

### Q4: 能修改已提交的预测吗？
**A**:
- 当前：不能（保持预测的诚实性）
- 未来：可能会添加修改功能

### Q5: 预测对我的学习有帮助吗？
**A**:
- **是的！** 研究表明：
  - 做预测 → 提升元认知
  - 元认知强 → 学习效率高
  - 学习效率高 → AI 使用更有效

## 🎓 使用建议

### ✅ 做法

1. **定期做预测**（每周或每个任务）
   - 频率越高，准确率越容易提升

2. **诚实地预测**
   - 不要预测"我会很完美"
   - 预测"我通常会怎样"

3. **记录任务细节**
   - Task ID 要清晰
   - 这样你能找到对应的实际任务

4. **认真验证**
   - 完成任务后，真的思考一下实际模式
   - 比对预测和实际

5. **定期查看准确率**
   - 看看趋势
   - 反思：你在改进吗？

### ❌ 避免

1. **随意预测**
   - "我随便选一个模式"
   - 这样数据没有意义

2. **忘记验证**
   - 预测后就忘了
   - 一定要事后验证

3. **预测后改变行为太多**
   - 预测是基于你的真实习惯
   - 别因为知道了预测就刻意改变
   - （除非你真的想改变）

4. **对模式定义理解模糊**
   - 再读几遍 Pattern A-F 的说明
   - 必要时查看 Patterns Page
