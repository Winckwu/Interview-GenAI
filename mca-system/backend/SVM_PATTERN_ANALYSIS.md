## 4. 误分类分析

### 唯一的严重误分类: D → A

```
误分的 2 个 D 样本都被分类为 A

根本原因分析:
─────────────

D 的典型特征:
  P1-P4: 1-2 (规划较少)
  M1-M3: 3   (验证严格)
  E1-E3: 3,2,3 (批判性)
  R1-R2: 2,1 (调整较少)
  总分: 26

A 的典型特征:
  P1-P4: 3,3,3,3 (系统规划)
  M1-M3: 3,3,3 (严格监控)
  E1-E3: 3,3,3 (深度评估)
  R1-R2: 3,2 (持续调整)
  总分: 35

测试集的这 2 个 D:
  可能具有较高的 P1-P4 分数
  → 导致更接近 A 的特征空间
  → SVM 决策边界将其分为 A

解决方案:
1. 确认这 2 个 D 样本是否真正代表 Pattern D
2. 或者，考虑降低 P (规划) 在 D 中的权重
3. 增加 M (监控) 的权重作为 D 的主要特征
```

### 边界误分类: C → A

```
误分的 1 个 C 样本被分类为 A

这是可以接受的误分类原因:
─────────────────────────────

Pattern C: 上下文敏感适应
- 特点: 平衡各维度,灵活调整
- 特征: 中等范围的多维评分

Pattern A: 战略分解控制
- 特点: 系统全面规划
- 特征: 高范围的多维评分

误分原因:
这个 C 样本可能:
- 在规划 (P1-P4) 上分数较高
- 在验证 (M1-M3) 上分数也高
- 在评估 (E1-E3) 上分数高
→ 导致整体高分,类似 A 的特征

这不是严重问题,因为:
✓ 这是特征空间的边界情况
✓ C 和 A 都代表"有效使用"
✓ 误分为 A 比误分为 F/E 好多了
```

## 8. 为什么 SVM 在这个任务上表现最好

### SVM 的优势

```
1️⃣ 特征空间分离明确
   - F 明确有 E2=0
   - A 明确是高维集中
   - E 明确有高 E2
   → SVM 容易找到线性/非线性分割面

2️⃣ 样本数量适中
   - 79 个样本对 SVM "恰好合适"
   - 足够学习模式,不足以过拟合
   - RF/XGBoost 容易在 79 个样本上过拟合

3️⃣ 支持向量集中
   - 每个模式都有清晰的"关键样本"
   - SVM 使用这些支持向量定义决策边界
   - 无需拟合 200 棵树,造成过拟合

4️⃣ 特征的"有缺失"特性
   - 特别是 E2 的"缺失"(F 的 E2=0)
   - 这种特性对 SVM 的 RBF 核很友好
   - 允许非线性的完美分割
```
